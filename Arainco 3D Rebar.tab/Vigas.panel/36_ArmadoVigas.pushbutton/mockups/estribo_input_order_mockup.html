Orden inputs — slot estribo

A ref B–E alt Viga demo: V-471546 · triple

Evaluar orden vertical de los tres grupos del controlador de estribos por viga (~360 px): Ext|Cent (ø, @, L), Confin. (escenario sección) y Suple inf. (Sí/No, ø, n). Cambie variante, suple y escenario desde la barra superior; marque pros/contras en el panel derecho.

Evaluación UX

A · Actual (referencia)

  • + Orden familiar — coincide con producción
  • + Zonas longitudinales (Ext/Cent) arriba, lógica de lectura izq→der
  • − Confin. y suple quedan «al fondo» — fácil olvidarlos
  • → Flujo: espaciar → confinar → opcional suple

B · Modelado primero

  • + Confin. primero — define patrón antes de @ por zona
  • + Alineado con «configurar → modelar»
  • − Rompe paralelismo visual Ext|Cent como primer gesto
  • → Flujo: sección → espaciar → suple opcional

C · Global arriba

  • + Reglas globales (confin + suple) visibles de entrada
  • + Compacto en altura si Confin|Suple comparten fila
  • − Fila superior muy densa en ~340 px
  • → Flujo: tramo/global → detalle por zona

D · Flujo longitudinal

  • + Sigue capas físicas: estribos → suple → confin sección
  • − Confin. al final contradice «modelar sección antes»
  • − Suple entre estribos y confin mezcla dominios
  • → Flujo: alzado → capa suple → patrón transversal

E · Compacto 2 bloques

  • + Solo 2 secciones — menos saltos visuales
  • + Confin. inline bajo Ext|Cent mantiene contexto
  • − Suple ancho completo añade altura al bloque 2
  • → Flujo: estribos+confin → suple dedicado

Notas flujo configure → model

Configure → model: el usuario define reglas de sección (confin.) y opciones de tramo (suple) antes de generar geometría. Variantes que suben Confin. o agrupan globals (B, C) encajan mejor con ese flujo. Variantes que priorizan @ por zona (A, D) encajan con edición iterativa de estribos ya modelados.
